Executive Summary

Two MoE architectures trained on identical sorting task. Same expert count (8), same top-K (3). Different routing mechanisms produce fundamentally different expert utilization patterns. Tested across 4 batch sizes (1, 5, 20, 64) and 3 sequence lengths (64, 128, 256).

🔑 Key Insight

BiBo achieves BOTH better balance AND better loss. At seq128, BiBo's Skywork normalization + bias updates produce near-perfect load balance (Gini=0.019, balance_ratio=0.88) while Qwen's aux loss actually results in worse balance (Gini=0.076, balance_ratio=0.68). Combined with 2.5× lower final loss, BiBo's routing is strictly superior on this task.

🔒 Routing Stability

How consistent are routing decisions across different inputs? Jaccard similarity of expert sets for 20 samples.

BiBo (avg: 0.31)

Lower stability = more adaptive. Conv router makes context-dependent decisions — same token gets different experts depending on neighbors. Scores: seq64=0.31, seq128=0.30, seq256=0.32.

Qwen3MoE (avg: 0.47)

Higher stability = more predictable. Linear router sees only current token → same embedding always routes same way. Scores: seq64=0.44, seq128=0.47, seq256=0.49. Better for debugging but less adaptive.

⚡ Expert Switching Rate

How often does the top-1 expert change between consecutive tokens? High = position-sensitive routing.

Interpretation

Switch rate near random (7/8 = 0.875) = no local coherence. Below random = "sticky" routing (same expert handles consecutive tokens). BiBo's conv router creates local coherence → lower switch rate.

🎭 BiBo Special Expert Analysis

BiBo's architectural innovation: Identity, Zero, and ReLU² experts alongside standard MLPs.

What Each Special Expert Does

ExpertOperationUsagePurpose
Identityoutput = input~16%Learned skip connection. Token already has good representation — no transform needed.
Zerooutput = 0~19%Dump bucket. Only shared Conv1D expert contributes. Saves compute for "easy" tokens.
ReLU²ReLU(x)²~14%Sparse activation. Produces sharper, more selective features than SwiGLU.

💡 Why This Matters

Qwen forces ALL tokens through identical SwiGLU MLPs. BiBo lets the router choose the right tool: full MLP for complex tokens, Identity for already-good representations, Zero for tokens where only local context (shared Conv1D) matters. This is why BiBo achieves lower loss with fewer parameters.

⚖️ Final Verdict

BiBo Router

✅ Strengths

  • Context-aware routing (Conv, k=3)
  • Meaningful expert diversity
  • Skywork norm prevents collapse
  • No aux loss needed
  • Zero/Identity = free skip connections
  • Lower final loss (0.1 vs 0.25)
  • Better balance at seq128+ (Gini=0.019)
  • Balance ratio 0.88 (vs Qwen's 0.68)

❌ Weaknesses

  • Lower stability (0.31 vs 0.47)
  • Conv incompatible with seq parallelism
  • Heuristic bias needs tokens to converge
  • Seq64 balance weaker (Gini=0.07)

Qwen3MoE Router

✅ Strengths

  • High stability (predictable)
  • Simple, parallelizable
  • Proven at scale (production)
  • Continuous gradient signal (aux loss)

❌ Weaknesses

  • No context awareness
  • Homogeneous experts only
  • Aux loss interferes with task loss
  • Worse balance than BiBo (Gini=0.076)
  • Higher final loss (0.25)
  • No "cheap" expert option
  • Persistent favorites (balance_ratio=0.68)
